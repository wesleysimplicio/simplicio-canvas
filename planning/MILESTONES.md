# Milestones

## M1 — Foundation: trustworthy architecture explorer

Goal: turn the current concept into a stable local-first repository explorer. Exit: a user opens a Python/TypeScript folder, sees accurate layered pieces and dependencies, filters/searches, navigates large graphs, and exports/imports a versioned snapshot. Target: v0.2.0. Tasks: SC-001–SC-006.

## M2 — Compose: safe visual programming

Goal: allow puzzle operations to describe real, reviewable code changes. Exit: create/move/connect/rename operations produce a deterministic plan and diff preview; invalid architecture is rejected; apply is explicit and undoable. Target: v0.3.0. Tasks: SC-007–SC-012.

## M3 — Embed: VS Code and Cursor

Goal: make the canvas a daily editor surface. Exit: extension opens the current workspace, syncs selections both ways, refreshes incrementally and applies approved plans through a guarded workspace service. Target: v0.5.0. Tasks: SC-013–SC-017.

## M4 — Intent: AI-native programming

Goal: move from manual assembly to intent-driven architecture transformations. Exit: AI explains the current graph, proposes typed operations, simulates impact, generates tests, executes only after approval and records evidence. Target: v1.0.0. Tasks: SC-018–SC-022.

Recommended execution order is the numeric task order; only tasks marked `ready` in `backlog.json` may start. Luna Medium should take one task per run.
